Overview

The goal is a platform, not only a graph runtime.

Earlier frameworks organized prompts, tools, and model calls. AgentWorld assumes the worker below the system is already a strong agent with tools, files, sessions, and long execution traces. This repository focuses on the platform foundation that makes a broader ecosystem possible.

Platform Foundation

Controllers, operators, A2A, runtime state, graph execution, and recoverability form the common base layer for strong-agent systems.

Skill Platform Seed

Skills can already be packaged and loaded per node. That creates the starting point for reusable capabilities, references, scripts, and future exchange.

Benchmark + Evaluation

A real platform also needs evaluation primitives, benchmark tasks, scorecards, and eventually public leaderboards for strong-agent workflows.

Application Ecosystem

Auto research, reliable agentic systems, agentic RL, and domain-specific workflows sit above the common platform instead of being hard-coded into the runtime layer.

Foundation Architecture

One platform foundation, four hard boundaries.

This page focuses on the currently implemented foundation layer. Provider complexity stops at the controller. Collaboration semantics live in the protocol. The graph runtime stays one subsystem inside the larger platform foundation.

01

Controllers drive real agents.

Provider-specific adapters create or resume sessions, map tool policies, parse event streams, preserve raw traces, and translate failures into runtime-safe signals.

Claude Code Codex OpenClaw
02

Operators expose one execution contract.

Each operator receives an objective, a state view, an inbox, artifacts, working directory rules, and session policy. It returns normalized outputs that the graph can route without special cases.

Objective Inbox Artifacts
03

A2A protocol keeps collaboration explicit.

Message payloads, handoffs, tool outputs, and artifact references move through a typed channel. Shared state and inter-agent communication remain separate on purpose.

Messages Tools Handoffs
04

The graph runtime is one foundation subsystem.

Graph compilation, routing, checkpoint persistence, resume, interrupt, and trace inspection stay in the runtime. Nodes stay small. Runs stay inspectable. This matters a lot, but it does not define the full platform by itself.

Foundation Runtime

What the platform foundation needs next.

The execution model is narrow enough to stay implementable and strong enough to grow into the shared operating layer below a larger strong-agent ecosystem platform.

1

Unified operator request

Objective, state slice, inbox, artifacts, working directory, tool policy, and timeout live in one request.

2

Normalized operator result

Status, messages, state patch, artifact outputs, handoffs, metrics, and trace references come back in one shape.

3

Persistent execution state

Checkpoint storage keeps every run resumable and makes interruption, replay, and audit practical instead of aspirational.

4

Composable graph semantics

Edges describe handoffs, branching, and review loops without leaking provider-specific control into the graph definition.
